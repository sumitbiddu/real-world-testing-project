# Automation Framework Overview

## Tools Used
- Selenium WebDriver
- Cucumber (BDD)
- Java (OOP with POM structure)

---

## Framework Design

The automation framework follows Page Object Model (POM) design pattern to improve maintainability and reduce code duplication.

---

## Covered Scenarios

- Login functionality
- Basic navigation flows
- Regression-critical paths

---

## Approach

Test scenarios defined in manual testing are mapped into automation scripts to ensure repeatability and regression coverage.

---

## Notes

Automation is focused on stable, high-priority flows rather than attempting to automate everything. This ensures maintainable and scalable test coverage.
